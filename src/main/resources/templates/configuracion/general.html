Control Financiero y de Caja

Configuraciones de seguridad para el manejo del dinero

Seguridad de Caja
Si está activado, NO se puede vender sin haber abierto caja al inicio del día.
Recomendado: SÍ (Control financiero estricto)
Si está activado, el cajero NO ve cuánto debería haber en caja al cerrar.
Obliga a contar el dinero real. El sistema calculará sobrante/faltante.
Recomendado: SÍ (Evita ajustes fraudulentos)
Límites de Efectivo
El sistema mostrará una alerta cuando el efectivo en caja supere este monto.
Recomendado: S/ 2000.00 (riesgo de robo)
Cuando se alcance el límite, el cajero debe depositar el excedente.
Buena Práctica:
• Abrir caja con monto inicial (S/ 100-200 para cambio)
• Depositar excedentes cuando se alcance el límite
• Cerrar caja contando el efectivo real

Facturación Electrónica (SUNAT)

Configuración para integración con proveedores de facturación electrónica

Importante: Esta configuración requiere contratar un servicio de facturación electrónica (OSE/PSE) como Nubefact, FacturadorSUNAT, Greenter, etc.
Configuración del Proveedor
URL del API del servicio de facturación contratado
Token de autenticación del servicio (mantener secreto)
NO = Modo Pruebas (Beta SUNAT - No genera comprobantes reales)
SÍ = Modo Producción (Facturas válidas ante SUNAT)
Certificado Digital (Opcional)

Solo si facturas directamente a SUNAT sin intermediario

Ruta absoluta al archivo de certificado digital
Contraseña para acceder al certificado digital
Recomendación:
Para PyMES, es más fácil usar un proveedor (OSE) como:
• Nubefact
• Facturador SUNAT
• Greenter
Ellos manejan el certificado y la conexión a SUNAT.
Próximos Pasos después de configurar:
1. Probar conexión con el proveedor (modo pruebas)
2. Verificar que se generen XML correctamente
3. Activar modo producción solo cuando todo funcione
4. Monitorear el primer día de facturación real
